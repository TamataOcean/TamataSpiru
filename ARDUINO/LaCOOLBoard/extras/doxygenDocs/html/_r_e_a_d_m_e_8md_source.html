92 Configuration Files :
93 
94 -The CoolBoard API heavily uses the SPIFFS for storing and retreiving configuration and data files
95 
96 This is a description of the configuration files and what are they used for :
97 
98  1/coolBoardConfig.json :
99 
100  logInterval: The time Interval to wait,in seconds, between two logs
101 
102  ireneActive: Put this flag to 1(true) if you are using the Irene3000 module
103 
104  jetpackActive:Put this flag to 1(true) if you are using the Jetpack modue
105 
106  externalSensorsActive: Put this flag to 1(true) if you are using a supported external Sensor
107 
108  sleepActive: Put this flag to 1(true) if you want your CoolBoard to enable Sleep mode
109  In Sleep Mode : your CoolBoard will do the following : -readSensors
110  -activate Actors(if any)
111  -log the data
112  -check for updates
113  -go to sleep for logInterval period of time
114 
115  userActive:Put this flag to 1(true) if you want your CoolBoard to collect userData : userName
116  CoolBoard MAC Address
117  TimeStamp
118 
119  manual /!\ : Put this flag to 1(true) to control to turn on/off your actors and bypass your initial actors configuration
120 
121  Be Careful with this mode!!.
122 
123  When this mode is actif , the CoolBoard will not restart automatically to apply any new configuration sent via
124  the update method.
125 
126  Restarting the coolBoard when in manual mode will deactivate all the actors!
127 
128  Never forget to reset this to 0(false) to go back to normal mode!

132  2/coolBoardLedConfig.json:
133 
134  ledActive: Put this flag to 1(true) if you want to have Light Effects from the on Board LED
135

137  3/coolBoardSensorsConfig.json:
138 
139  temperature : Put this flag to 1(true) if you want to collect the Temperature using the BME280 Sensor
140 
141  humidity : Put this flag to 1(true) if you want to collect the humidity using the BME280 Sensor
142 
143  pressure : Put this flag to 1(true) if you want to collect the pressure using the BME280 Sensor
144 
145  visible : Put this flag to 1(true) if you want to collect the visible light index using the SI114X Sensor
146 
147  ir : Put this flag to 1(true) if you want to collect the infrared light index using the SI114X Sensor
148 
149  uv : Put this flag to 1(true) if you want to collect the ultraviolet light index using the SI114X Sensor
150 
151  vbat : Put this flag to 1(true) if you want to collect the battery voltage
152 
153  soilMoisture : Put this flag to 1(true) if you want to collect the soil Moisture
154 
155 
156  4/externalSensorsConfig.json:
157 
158  sensorsNumber: the number of supported external sensors you connect to the coolBoard
159 
160  reference: the reference of a supported external sensor(e.g NDIR_I2C , DallasTemperature )
161 
162  type: the type of the measurments you are making (e.g : co2, temperature,voltage ... )
163 
164  address : the sensor's address , if it has one (e.g : NDIR_I2C CO2 sensor's address is 77 )
165 
166 
167  5/irene3000Config.json:
168 
169  waterTemp.active: Put this flag to 1(true) in order to use the temperature sensor connected to the Irene3000
170 
171  phProbe.active: Put this flag to 1(true) in order to use the ph sensor connected to the Irene3000
172 
173  adc2.active: Put this flag to 1(true) in order to use the extra ADC input of the Irene3000
174 
175  adc2.gain: this is the value of the gain applied to the extra ADC input of the Irene3000
176 
177  adc2.type: the type of the measurments you are making (e.g : co2, temperature,voltage ... )
178 
179 
180  6/jetPackConfig.json and coolBoardActorConfig.json:
181 
182  Act[i].actif: Put this flag to 1(true) in order to use the jetpack output Nï¿½i (0..7)
183 
184  Act[i].inverted:Put this flag to 1(true) if the actor is inverted (e.g : a cooler is activated when Temp>TempMax)
185  Put this flag to 0(false) if the actor is notInverted(e.g : a heater is activated when Temp<TempMin)
186 
187  Act[i].temporal:Put this flag to 1(true) if you want the actor to be actif of a period of time ,
188  then inactif for another period of time.
189 
190  Act[i].type:["primaryType","secondaryType"] : this array contains the priamryType and the secondaryType of the actor
191  -The primaryType is the type associated to the sensors.
192  (e.g : primaryType : "Temperature" is associated to the sensor of type "Temperature").
193  -It can also be empty , in case of PURE temporal actors
194 
195 
196  -The secondaryType is only used in temporal mode.
197  it can be : -"" (empty):the actor will be on for a period of timeHigh ms
198  the actor will be off for a period of timeLow ms
199 
200  -"hour" :the actor will be on when the Hour is equal or greater then hourHigh
201  the actor will be off when the Hour is equal or greater then hourLow
202 
203  -"minute":the actor will be on when the Minute is equal or greater then minuteHigh
204  :the actor will be off when the Minute is equal or greater then minuteLow
205 
206  -"hourMinute":the actor will be on when : Hour == hourHigh AND Minute >= minuteHigh
207  Hour > hourHigh
208  the actor will be off when : Hour == hourLow AND Minute >= minuteLow
209  Hour>hourLow
210 
211  /!\ NOTE that if both primaryType and secondaryType are valid : the actor will be in
212  mixed mode : it will need to valid both Time and measurment conditions to go on or off
213 
214  Act[i].low:[rangeLow,timeLow,hourLow,minuteLow] : this array contains the values previously described:
215  -rangeLow is the minimum of the range at which
216  to activate(deactivate in inverted mode) the actor
217 
218  -timeLow is the time spent off in temporal mode
219 
220  -hourLow is the hour to turn off the actor when secondaryType is hour or hourMinute
221 
222  -minuteLow is the minute to turn off the actor when secondaryType is minute or hourMinute
223 
224 
225  Act[i].high:[rangeHigh,timeHigh,hourHigh,minuteHigh]: this array contains the values previously described:
226  -rangeHigh is the maximum of the range at which
227  to deactivate(activate in inverted mode) the actor
228 
229  -timeHigh is the time spent on in temporal mode
230 
231  -hourHigh is the hour to turn on the actor when secondaryType is hour or hourMinute
232 
233  -minuteHigh is the minute to turn on the actor when secondaryType is minute or hourMinute
234 
235 
236  Note that the coolBoardActorConfig.json contains only one Actor.
237 
238  7/mqttConfig.json:
239 
240  mqttServer: This is the mqttServer (ip/url) address
241 
242  user: This is the userId
243 
244  bufferSize: This is the memory allocated to the mqtt buffer in bytes
245 
246  inTopic : this is the topic that the coolBoard subscribes to (receives updates from )
247 
248  outTopic : this is the topic that the coolBoard will publish to.
249 
250 
251  8/rtcConfig.json:
252 
253  timeServer: NTP server ip address
254 
255  localPort: port used to make the NTP request to update the time
256 
257 
258  9/wifiConfig.json:
259 
260  wifiCount: the number of wifis saved in this configuration file
261 
262  timeOut:access point timeout in seconds.
263 
264  nomad: put this flag to 1(true) to activate nomad mode.
265  in nomad mode : the coolBoard will only try to connect to known WiFis.
266  if it fails it will NOT lunch the access point.
